Overview

Core Concept

Welcome to the StyleSite CSS framework specifications. StyleSite is a pure frontend layout library engineered to output typography arrays entirely unboxed on the viewport canvas. It removes side-by-side table grids, borders, and card shapes, presenting content linearly on a vertical axis using a strict one horizontal component per row stack rule to ensure flawless legibility.

Linear Axis Flow

By forcing text blocks and sections to cascade individually down the screen layer, the framework eliminates grid formatting limitations. Content blocks fluidly adjust to fit mobile screens with absolute zero risk of compression or text overlaps.

Installation

Global CDN Link

To drop StyleSite directly into any project, simply import the production CSS file reference into the document <head> module:

<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/sahirpunjwani/stylesite@main/stylesite.css">

Local File Inclusion

Alternatively, copy the stylesite.css file straight into your local workspace directory and reference it natively:

<link rel="stylesheet" href="stylesite.css">

Class Usage

1. Workspace Activation

Normalize your workspace environment by adding the core .ss-workspace class rule directly to the parent <body> tag:

<body class="ss-workspace">
    <!-- Layout segments cascade natively -->
</body>

2. Single-Row Text Stacking

To align typography blocks cleanly without nested containers, pass rows inside the .ss-row-stack layout token element wrapper:

<div class="ss-row-stack">
    &div class="ss-unboxed-content zm-motion-reveal">
        <h3>Section Heading Title</h3>
        <p>Your description text outputs and spaces itself out automatically...</p>
    </div>
</div>
